  * fix(hooks): v5.143.33 — after hooks are timer-only; remove spurious dispatch-path probe (StoneCypher/fsl#1327)
  * The dispatch path probed _after_hooks with the transition target (or action
name), so entering a hooked state — or a same-named action — spuriously fired
the after hook, and a leave-and-return before the timer elapsed fired it twice
for one elapse. After hooks are the after-timer's companion and fire only from
the state-timeout path; the dispatch probe is removed. Adds three fsl#1327
regression tests and pins the after-hook benchmark to carrying-cost (expectFire
false). Rebased onto current main and rebuilt at 5.143.33.

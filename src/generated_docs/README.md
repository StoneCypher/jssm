# generated_docs

Build-regenerated documentation artifacts.  **Do not hand-edit anything here
except this README** — the `perf_chart` step of `npm run build` rewrites these
files from the `perf_results` data branch on every full build, so every
release PR carries the freshest snapshot through the normal pipeline.

- `perf_chart.svg` — the graviton performance trend: one log-scale panel per
  benchmarked operation, one line per machine shape, PRs ordered by number
  then releases by semver.
- `perf_data.json` — the same data machine-readable: every measured run's
  identity, version, date, and full benny results, in chart order.

Regeneration is **deterministic**: artifacts are stamped from the newest data
point's date (never wall clock) with stable ordering, so a build over
unchanged perf data produces byte-identical files and no diff churn.  When
`perf_results` is unreachable (offline builds), the step warns and keeps the
existing files.

Generated by [`src/scripts/make_perf_chart.cjs`](../scripts/make_perf_chart.cjs)
(#748), which also posts a fresh chart comment to the perf-tracking issue
whenever a graviton run publishes new data (`.github/workflows/perf_chart.yml`).

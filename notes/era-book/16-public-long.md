**The registry, loop-first.** *Manager:* Before building any marketplace: prove the research-
grade core on a tiny vertical — an agent states a contract, queries, assembles a composite,
gets a green proof or a minimal counterexample, swaps a part under refinement. If that loop
isn't cheap enough for an agent's inner loop, the thesis is hollow; prove it before indexing
anything. *CS:* slice per registry-close: TS/JS only, hierarchical operator only, checkable
band only, author-claimed + one registry-verified(ts-js) path, counterexample traces,
adapter-as-proof-carrying-component, skeletal third-party adjunct binding (namespace+hash);
THEN trust tiers (target-qualified, never bare), competing adjuncts, policy-constrained
resolution (SAT/CSP), behavioral semver via proof-dependency invalidation.

**Guardrails-as-product.** *Manager:* The keynote demo becomes a product: an agent's tool
surface gated by a certified machine — "never charge twice without confirm" as a proof, not
a prompt. FSL securing the interface that exposes FSL to AI. *CS:* `mcp --policy guard.fsl`;
tool calls = events, legality = current-state transition check; policy machines certified +
lockfile-pinned; policy decisions land on the tape (auditable); template library
(no-network-after-secret, approval-before-delete, tests-before-deploy).

**`query` + semantic patches.** *Manager:* Fleet governance — one behavioral question across
every machine in an org, every hit shipping its witness tape — and the structured-edit
protocol that lets many agents work one machine without trampling text. *CS:* `query`
predicates (reachable/flow/through/population/edition) over the checker machinery fleet-wide;
witness tape per hit; semantic patches (§29 door): fsl.patch.v1-style ops, source-hash
preconditions, dry-run + behavioral-diff receipts; the git semantic merge driver rides the
same machinery.

**infer + synth (the aspirational bookends).** *Manager:* Learn a draft machine from
production logs; derive a guardrail machine from properties, correct by construction. The
ladder completes: infer learns from behavior, check judges against properties, synth derives
from properties. *CS:* infer = passive L*-flavored automata learning over tapes/logs →
draft + `minimize` cleanup; synth = safety-game solving over Dwyer-pattern specs (decidable
safety fragment) → maximally-permissive policy + certificate; both stay labeled aspirational
until the eval bar clears.

**The corpus at scale.** *Manager:* The manual/curriculum/eval program full-size: the book
with its v11-unlocked spine, learn-by-repairing as both on-ramp and eval set, the taxonomy's

manual-templates spec; every instance = NL↔FSL corpus pair, canonically formatted.

**The distribution program.** *Manager:* Make FSL visible where humans and models look: MCP

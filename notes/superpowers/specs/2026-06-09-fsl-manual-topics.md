## Cross-cutting principles to thread through the WHOLE manual
These recur everywhere; explain them once, early, and refer back:

- **Bounded by default, opt into unbounded knowingly.** Finite types, bounded containers/strings, tape retention, the microstep bound ‚Äî every default is the safe/analyzable one. *Why:* keeps machines verifiable and memory-safe; going unbounded is always a visible act.
- **The `finite`/checkable tier is the verification substrate.** Three bands: small-finite (enumerate), large-finite (SMT), infinite/rich (bounded analysis only). *Why:* finiteness is what makes model-checking decidable. The `finite` attribute *enforces* the line.
- **Determinism & replay: input is the single source of truth.** Seeded RNG + recorded input (events *and* sensor reads) ‚Üí everything else regenerates. *Why:* reproducible runs, portable repro bundles, counterexamples-as-tapes.
- **Total expressions.** No loops/recursion/effects in expression position; iteration only via bounded HOFs. *Why:* every expression terminates; guards always have a value.
- **Host-agnostic.** Spec defines semantics; per-host runtime; a conformance suite is the executable definition; never borrow host behavior. *Why:* one `.fsl` runs identically on JS/Rust/C/Erlang/‚Ä¶.
- **The pinning principle.** Pin anything not uniquely determined by input ‚Üí identical cross-host output: hash *algorithm*, compression *encoder*, Unicode *version*.
- **Behavioral (source) vs observational (API).** Source *defines* what the machine is (`assign`/`where`/entry/exit/handlers); hooks *observe* what it does. *Why:* keeps `.fsl` files meaningful and verification clean.
- **Units = dimensions + nominal branding.** Dimensional analysis catches `m/s` errors; branding catches `user_id` vs `order_id` (and torque vs energy). You need both.

## Part 0 ‚Äî The on-ramp
- **Learn-by-repairing (build the manual's intro track this way).** Each lesson is a deliberately broken machine + its counterexample tape; the reader fixes until `check` passes. Graded ladder: syntax error ‚Üí wrong guard ‚Üí missing transition ‚Üí contract violation ‚Üí temporal-property violation ‚Üí composed-system route bug. *Why:* it teaches the repair loop AS the learning loop, works identically for humans and models, generates the bug-fix eval corpus as a byproduct, and every lesson is doc-tested by construction. *Pair each lesson with the certified-stdlib part it ends up rebuilding* (lesson 4 ends with "you just built the circuit breaker ‚Äî here's the certified one").
- **The certified stdlib as teaching vocabulary.** Introduce debounce / retry-with-backoff / circuit breaker / rate limiter / saga / turnstile early and reuse them as the running parts-box; teach reading an interface card before teaching writing a machine.

## Part I ‚Äî Foundations
- **What FSL is now.** A state machine that is *also* a typed data machine, a transducer (I/O), and a verifiable/testable model. The `prop` (where you are) / `val` (how you got there) / `sensor` (what's out there) trichotomy. *Gotcha:* this is a major-version leap; set expectations. (megaspec ¬ß1)
- **GoF design patterns as *features* (positioning/teaching asset).** ~10 of the 23 Gang-of-Four patterns aren't *implemented* in FSL ‚Äî they **are** language features: **State** (the machine + function-typed props), **Strategy** (function-typed `val`), **Observer** (hooks), **Memento** (serialize / snapshot / tape), **Command** (typed-payload events + tape), **Visitor** (`case` over ADTs), **Mediator** (`System` + `wire`), **Chain of Responsibility** (transition resolution order), **Composite** (state groups), **Decorator** (hooks-as-decorators). ~6 more are naturally expressible (Iterator/Interpreter/Adapter/Proxy/Builder/Prototype); the creational OO-instantiation ones (Factory Method / Abstract Factory / Singleton) are out of a state machine's domain unless actor-spawn is added. *Teach:* lead with "you don't *write* these patterns, you write states" ‚Äî State/Strategy/Observer/Memento first. (analysis 2026-06-10)
- **The automata zoo ‚Äî FSL models many formalisms as feature *configurations* (a positioning chapter, sibling to the GoF one).** From **Systems + wiring:** protocols, **session types**, CFSMs, I/O & interface automata, **workflow/BPMN/DMN**, cellular automata, synchronous languages (Esterel/Lustre). From the **model-class ladder:** regex, CFG-parsers/RSMs, visibly-pushdown/nested-word, Petri, tree automata, attribute grammars. From **weighted edges:** Markov chains/MDPs, weighted automata, HMMs, reward machines. From the **transducer model:** Mealy/Moore, FSTs, tree transducers. From **vals-as-registers:** register/data automata. From the **verifier:** Kripke structures/LTS, B√ºchi. From **groups+history:** statecharts/UML/SCXML. *Meta to lead with:* these are **coordinates** in a feature space (memory √ó communicating-copies √ó weights √ó I/O-shape √ó registers) ‚Äî **recovered, not implemented** (same framing as "GoF patterns are features"). *Name the honest non-fits:* timed automata/PTA (deferred), hybrid automata (continuous ODE ‚Äî off-model), œÄ-calculus mobility & unbounded channels (the totality ceiling). (megaspec ¬ß17 catalog) *Worked example to anchor it:* BPMN's parallel gateway ‚Üí why you need rung-2 tokens (a plain FSM has one current state; tokens are a multiset). *Two more framings worth teaching:* FSL + contracts is a lightweight **executable / visual TLA+ / Event-B** (state + invariants + next-state); and the **inverse of verification** ‚Äî reactive/LTL **synthesis**, supervisory control, Angluin **L\*** learning ‚Äî *generates or infers* a machine from a spec/oracle. *Hot example to lead with:* **behavior trees** (game-AI / robotics = a `tree` ticked synchronously).
- **Migration.** `var`‚Üí`val`, `property`‚Üí`prop` (deprecated alias, **removed in v7**); the assignment verb is `assign` (because `set` is now the set *type*). *Gotcha:* hand-bumping is forbidden; releases are coupled to `main`.

## Part II ‚Äî Data & Types
- **`val` (extended state).** Declare, type, `default`, `required`, the `vals` config option; read `.val()`, write via `assign` (slot assignment `a[i]:=`, sequential, `old` pre-state). *Decision:* imperative/sequential assign; `old` is the pre-transition accessor. (¬ß5, ¬ß9)
- **`prop` (now typed).** State-bound constants, optional type annotations (backward-compat: unannotated stays legacy). (¬ß5)
- **The numeric tower.** `int` (s53), sized `int8..128`/`uint8..128`, `longint`, `float`/`double`, `decimal` (banker's rounding), bounded forms. *Decisions:* receiving-type promotion; overflow = error + opt-in saturate (no wrap); IEEE NaN + `isnan`/`isinf`/`isfinite`. *Gotcha:* bitwise on **sized** ints only, not bare `int`/`longint`. (¬ß4.1)
- **Strings.** Codepoint default, grapheme via `+` suffix, UTF-8 byte view, negative slicing. *Gotcha:* bounded `string len 0..N` for the finite tier. (¬ß4.1, ¬ß8)
- **Containers.** tuple/array/set/map/record (+ bounded `len`/`size`); **no `object`** (use ADTs + `any`). *Decision:* keys/set-members = number/string only; values = anything. (¬ß4.2)
- **ADTs (sum types).** `type Shape = circle(r) | rect(w,h)`; recursive ‚áí rich-tier. *Why:* pattern matching is built for them. (¬ß4.3)
- **Variants.** `option`, `any` (open variant, **sound-narrow-only** via `case`); `null`/`undefined` (JS semantics, distinct, forbidden in arithmetic, declared-nullable `T?`). *Gotcha:* `null = undefined` is false. (¬ß4.3‚Äì4.4)
- **Type aliases.** `type Celsius = int -273..1000`. (¬ß4.4)
- **Units (the big one).** Dimensioned types, the SI prelude (base units + prefix system ‚Üí `km`/`ms`/`ŒºA` free + derived units), declaring units (product *and* quotient), dimensional checking, auto-conversion, domain newtypes. *Decisions:* phantom types / **zero runtime cost**; in-scope (later phase). *Gotchas to teach with examples:* the km/h‚Üîm/s and bits‚Üîbytes wins; **torque ‚â° energy** (dimensional blind spot ‚Üí fix with branding); **affine** temperature (¬∞C/¬∞F vs K); `KB`(1000) vs `KiB`(1024); square dollars = variance, sqrt halves exponents. (¬ß4.5) *(Manual should reuse the worked examples: battery charger, cruise control, download manager, rocket controller, risk monitor.)*

## Part III ‚Äî The expression language
- **Operators & precedence.** Arithmetic (`mod`/`rem`/`div`), comparison (cross-type = error; total `compare`), the boolean zoo (`and`/`or`/`nand`/`xor`/`implies`/‚Ä¶ + short-circuit `andalso`/`orelse`), bitwise, `in`, `++`, `|>`. (¬ß6)
- **Control & matching.** `if/then/else` (expression), `let ‚Ä¶ in`, `case` (full pattern matching, destructuring, `when`, exhaustiveness + `else`), bounded quantifiers `forall`/`exists`. (¬ß6)
- **Container protocol & lambdas.** `map`/`filter`/`foldl`/`foldr`/`all`/`any`/‚Ä¶ ; bracket-index **throws**, `get` returns **option**; uncurried lambdas (inline checkable + first-class rich), by-value read-only capture. *Gotcha:* no `foreach` (use `fold`). (¬ß6)
- **Function-typed props & vals (a flagship teach point).** Because lambdas are first-class values, a `prop` or `val` can *be* a function. A function-typed **prop** = behavior that changes with state = the **State pattern**, native and verifiable (one lambda pinned per state). A function-typed **val** = a reassignable **carried strategy**. *How it stays sane:* function values **defunctionalize** to `(stable tag, by-value captures)` so they serialize / hash / replay / cross-host like any data ‚Äî there is no opaque host closure (this is *why* the totality + by-value-capture + host-agnostic-semantics constraints exist). *Gotchas:* equality is **intensional** (same code+captures, *not* same behavior ‚Äî two lambdas computing alike compare unequal); function-typed vals widen the verifier's state space (finite-checkable only if the assignable-lambda set + captures are finite-typed); editing a lambda body invalidates old repro bundles (tag = AST content-hash). (¬ß4.4 / ¬ß5 / ¬ß15)
- **Math & stdlib.** The builtin catalog; trig in **degrees** (with `deg`/`¬∞`/`rad`), RNG distributions; blessed infra: encoding (`base64`/`base64url`/`hex`/percent), pinned `hash`, compression (`zstd`+`lz4`, URL-safe via base64url). *Principle:* domain-neutral core; domain toolkits = userland. (¬ß7)

## Part IV ‚Äî Behavior, errors, lifecycle
- **Execution model ‚Äî macrostep / microstep / stable (THE keystone; teach it first in this Part).** A **macrostep** = one external stimulus run-to-completion; a **microstep** = one fired transition; **stable** = the quiescent config where no transition is enabled. *Why keystone:* invariant checks, val-commit, journal-rollback, post-macrostep hooks, observers, serialization, and the tape **all key off "stable."** *Teach the pipeline:* before-macrostep ‚Üí fire (exit ‚Üí `require` ‚Üí `assign` ‚Üí `ensure` ‚Üí entry ‚Üí raise) ‚Üí cascade eventless/internal ‚Üí stable (invariants ‚Üí commit ‚Üí at-stable). *Plus:* deterministic microstep order (priority + doc-order; nondeterminism only via seeded `probabilistic`); internal-before-external (SCXML FIFO); microstep bound faults runaway eventless; verifier proves quiescence for `finite`. (¬ß12)
- **Guards & contracts (keep the four apart).** `where` = enabledness (silent non-transition); `require` = caller obligation (misuse = error); `ensure` = machine obligation (with `old`); `invariant` = always-at-stable (RTC). *Violation model:* each is a **proof obligation** the verifier discharges **and** a runtime assert that **compiles out under `sealed`/proven**; contract exprs total/pure; **no inheritance**. (¬ß10, #1355)
- **The error model (teach the three surfaces).** in-machine `last_error` (option<Error>, sticky) + `error_count`; in-language composable `error` **tape**; host-side `rollback` **event** (carries the pre-error snapshot); behavioral **error-recovery transitions** (`require X else -> S`, `on error -> S`). *Decisions:* atomic rollback + notify; **no `on rollback` handler** (breaks atomicity); domain **error states** (`state X: { error; }`, local form of `failed_outputs`) ‚âÝ runtime faults. (¬ß11, ¬ß17)
- **Lifecycle & hooks.** Source: `on construct`/`destruct`/`entry`/`exit`, `on rejection`, `on any transition`/`any action`; **named handlers** (internal reusable effect blocks); **named hooks** (caller-provided callbacks the machine `call`s, optionally `required` = dependency contract). *Decision:* behavioral-vs-observational split (25 runtime hooks ‚Üí only ~4 lifecycle deserve source). *Gotcha:* observational hooks are pure observers; introspection (`has_hook`/`hooks_on`) is currently stubbed. *Lineage:* `hooks: open/closed` (fsl **#617**, grammar #644/compiler #645) = the open/closed hook surface; named hooks + `hooks: closed` *fulfill+extend* it (allowlist ‚Üí typed `required` contract ‚Üí codegen hook-interface export). **New from RTC:** macrostep-level hook points `before-macrostep`/`at-stable`/`post-commit` (the run-to-completion boundary points that didn't exist before). (¬ß12, ¬ß3)

## Part V ‚Äî I/O, tapes, systems
- **Reactive I/O.** Input via `on chan(payload)` (no `consume` ‚Äî input *drives* the machine); output via `emit chan <- v`; Mealy + Moore; named typed channels. (¬ß14)
- **Sensors & time.** Sensors = input-level dual of props (read-only, externally wired, recorded for replay); **time = a pluggable, caller-provided sensor** (default clock, tick source with `.advance`, custom), declared abstract model for verification. *Teach the shared-clock multi-machine simulation.* (¬ß14)
- **Tapes.** Input tape = the single retained source of truth (default retention 100,000, tunable/unlimited); output/log/error tapes **not retained** (regenerate by replay); `log`/`error` standard, separate from main output; tokenizer = composition; first-class tapes. (¬ß14)
- **Multi-machine systems.** `machine {}` blocks + `wire` + `import`; `Machine` (`from`/`sm`) vs `System` (`system_from`/`ms`); the system fns (`make_system`/`compile_system`/`serialize_system`/`deserialize_system`); per-machine API via `.machine(name)`; the web component auto-detects. *Decisions:* two types/no-union; sync shared-clock default; private+shared scoping. (¬ß14)
- **Three composition axes (teach *when to use which*).** **Group hierarchy** (`&group` ‚Äî one machine, shared tape/clock ‚Äî Composite), **machine-val embedding** (`val sub : machine T` ‚Äî by-value, owned, parent-driven, snapshotted-with-parent ‚Äî HAS-A; the Factory-pattern primitive), **system wiring** (`wire` ‚Äî peer machines ‚Äî Mediator). *Rule:* **embed when you own it, wire when they're peers, group when it's one machine's sub-states.** *Gotcha (the why):* embedding is an acyclic **tree** (by-value, no self-containment, rich-tier like recursive ADTs); wiring is a possibly-**cyclic graph** (A‚ÜîB handshakes) ‚Äî that topology difference is exactly why machine-vals **don't subsume** Systems; both still share one `Steppable`/`Serializable`/`Renderable` + columnar/journal/tape engine. (¬ß14)
- **Factories (a flagship chapter ‚Äî the factory_girl / stochastic-tester story).** A `required` val with no `default` **is** an unfilled parameter; the `factory` block binds vals + identity targets (`instance_name`/`machine_name`/`start_at`/`seed`); `with()` curries; `make()` vs `at(n)` (pure core, cursor sugar). *Teach with the RCT park:* a customer factory whose `name <- "Cust ${seq()}"` and `patience <- rand(20, 95)`. *Decisions to explain:* instance *n* of seed *s* is a **coordinate, not an anecdote** (replayable fixtures); construction-time `rand` keeps machines `finite`-eligible (over-approximation ‚Äî randomized fixtures, verifiable instances); unseeded = clock-seeds-once-at-root **and is recorded** ("I can't reproduce it" stops being a sentence). *Gotchas:* one factory block per document; bindings read earlier bindings only; host-callback bindings = non-portable/non-serializable; `UnboundParameterError` lists *all* missing names. (¬ß21)
- **The seed tree.** One root seed; everything below derives via `derive(parent, label)`; per-instance, per-call-site streams. *Teach:* parallel population builds with zero coordination; changing one binding never perturbs another's stream. (¬ß21.2)
- **Systems orchestration: populations, routes, lifecycle.** `many Customer via factory count n max 500`; routes (`on customers.left(s) -> gate 'depart'(s)`) vs plain `wire`; observation sugar (`enters`/`exits`/`takes` ‚Äî machines never know they're in a system); `on action` = the system's inbound surface; `where` selectors; `spawn`/`retire sender`. *Teach the dispatch model:* deterministic FIFO, document/write/creation order, system quiescence ‚Äî same seed + same stimuli = bit-identical run. *Gotchas:* `sender` is the only context name (no computed member refs); `seq()` indices never reused after `retire`; queue-depth bound is the system-level microstep bound; `on_undeliverable` defaults to **error** (loud) ‚Äî `dead_letter` is the inspectable opt-in. (¬ß22)
- **Systems in systems.** Member contract = actions in, channels out ‚Äî systems satisfy it too; child runs to quiescence, emissions bubble to the parent; **containment is acyclic (enforced), wiring may cycle** ‚Äî teach the difference. Now *four* composition axes with groups/embedding/wiring: **group when it's one machine's sub-states, embed when you own it, wire when they're peers, contain when you orchestrate a fleet.** (¬ß22.4)
- **Overlapping state groups.** `&group : [...]`, nest `&c` / spread `...&c`, **group config objects** (`&g : [members] { config };` or separate `&g : { config };` ‚Äî applied to all members, group-metadata tier of the cascade), the unified styling cascade, boundary hooks, per-group history. *Teach:* > hierarchical states, *represents* Harel (compact orthogonality = parallel regions, deferred). *Gotcha:* it's `:` not `=`; `[‚Ä¶]`=members vs `{‚Ä¶}`=config; state-specific config beats group; nearer/deeper group wins. (¬ß19)
- **Transitions: multigraph & wildcards.** Default 1 edge; **action names** to get more; ‚â§1 unlabeled edge (= what `transition(to)` takes); probabilistic edges are the rolled exception; wildcard `*` source + `'*'` action. *Teach the resolution order:* specific-action > wildcard-action > unlabeled > wildcard-source > rejection. **Probabilistic list target** `a P% -> [b c]` ‚Äî list targets *already* fan out (`ArrowTarget ‚äá LabelList`). **Model:** origin-`%` = the **group's** weight; optional per-member `%` *inside* the list = within-group shares; multiply (`a 50% -> [b 20% c 80%]` ‚Üí 10%/40%; inner omitted ‚Üí uniform). *Teach:* syntax-compatible with old `P% -> [list]`; the group keeps its stated weight (vs the old copy behavior, which differed only with sibling edges). Full control via inner weights. *Same weighted-`LabelList` as `start_states` ‚Äî cross-ref Weighted start states; teach them together as one construct in two positions.* (decision log)

## Part VI ‚Äî Verification & testing (a flagship part of the manual)
- **The verifiability tiers & `finite`.** What's allowed/forbidden; the capability/**disallow** attributes & profiles (`finite`/`pure`/`sealed`/`deterministic`/`stateless`); each buys docs + checker assumption + runtime specialization. **`finite` vs `checkable` (must spell out explicitly):** `finite` = the hard *enumerable* line (small-finite, exhaustible); `checkable` = the softer *"the checker may attempt it"* tier (large-finite/SMT). They are **different attributes** and the manual must teach the boundary with concrete "this is finite / this is only checkable / this is rich" examples. (¬ß3)
- **The automata ladder & the model-class attribute.** A machine declares its rung ‚Äî `regular`/`pushdown`/`petri`/`tree` ‚Äî which sets the decision procedure *and enforces* the decidability tripwire (pushdown: **one** push/pop stack, no second stack; petri: monotone counters, **no zero-test**; tree: finite carrier). *Teach:* the val type *is* the auxiliary memory, the rung *is* the language class, totality caps below Turing. **`tree`** gives **verifiable validators / type-checkers** (recognizers) *and* **tree transducers** = **verifiable, schema-preserving compiler passes** (the basis for transcompilation, #4). *Gotcha worth a callout:* dropping the zero-test / the second stack isn't pedantry ‚Äî it's the exact line between decidable and Turing. (¬ß3/¬ß17)
- **Petri nets ‚áÜ VASS (the `petri` / rung-2 model class) ‚Äî teach both names, and that they're the *same machine*.** A **Petri net** is the token/graphical view (places hold tokens; transitions consume/produce; a transition fires only if its input places have enough tokens). A **VASS** (*Vector Addition System with States*) is the counter/automata view: a finite-state control + **k non-negative integer counters**, where each transition adds a fixed integer **vector** to the counters and is enabled only if they stay **‚â• 0**. They're inter-translatable ‚Äî **places ‚Üî counters, firing ‚Üî vector addition, token-availability ‚Üî non-negativity**. *The point for FSL:* **rung 2 literally *is* a VASS** ‚Äî "a state machine **+ a `bag` val**" = finite control + counter vector; `bag[x] += 1`/`-= 1` are the vector adds, `>=` guards are the non-negativity constraint. (Petri nets are how you *draw* it; VASS is how it's *built*.) *Decidability to teach:* **coverability** ("can a counter reach/exceed N?") is decidable, EXPSPACE; **reachability** is Ackermann-complete; and the **no-zero-test** rule is load-bearing ‚Äî a zero-test turns it into a Minsky counter machine ‚Üí Turing ‚Üí undecidable, which is why `petri` *forbids* it. *Worked example:* producer/consumer with an unbounded buffer. (¬ß3/¬ß17)
- **The property-pattern catalog (teach BEFORE raw temporal logic).** Dwyer patterns as the front door: response, precedence, absence, universality, existence ‚Äî each √ó scope (globally / before / after / between). *Teach:* choose a pattern, fill the holes; raw LTL is the escape hatch, not the entry; `lint --suggest-properties` proposes candidates from structure. *Why:* ~92% of real-world specs fit the catalog, and miswritten `eventually` formulas are the classic verification footgun ‚Äî for humans and models alike. (¬ß17)
- **Refinement & safe evolution.** `diff --behavioral` (v2 simulates v1) + refinement certificates; teach the upgrade story: prove compatibility, don't test for it. *Gotcha:* refinement is over the *old alphabet* ‚Äî new behavior on new actions is free. (¬ß25)
- **Temporal properties.** Safety (`always`/invariants) vs liveness (`eventually`/`leads-to`/`until`/`weak-until`/`always eventually`/‚Ä¶), past operators (`once`/`since`), bounded (`within N`), probabilistic (PCTL). *Teach:* counterexamples are **replayable tapes** (finite trace / lasso). Dense-time (#1361) and PTA (#1362) are advanced/deferred. (¬ß17)
- **Weighted start states.** `start_states: [a 30% b 70%]` = initial distribution Œº‚ÇÄ; companion to weighted transitions ‚Üí full Markov chain. **Unifying frame (lead with this):** it's the *same weighted-`LabelList`* as the probabilistic list target, seen from **two positions** ‚Äî an **arrow target** has an origin, so two levels (group `%` √ó member `%`); a **start set** has no origin, so one level (members *are* Œº‚ÇÄ). One rule everywhere: members carry optional `%`, uniform if omitted, normalized in context. (¬ß17)
- **Random walks & statistical MC.** Seeded reproducible walks; statistical model checking (works on rich-tier); coverage; rare-event/importance sampling. (¬ß17)
- **Scaling the checker (teach the three-rung ladder explicitly).** Native exhaustive (small machines) ‚Üí **assume-guarantee compositional** (contracts compose, state spaces don't ‚Äî teach with the 500-customer park: one contract proof, not a 500-fold product) ‚Üí **CEGAR** (aspirational; abstract, then refine where the spurious counterexample lied). Also the *direction* ladder: `infer` learns from behavior, `check` judges against properties, `synth` derives from properties. (¬ß23/¬ß28)
- **Flow contracts (the security chapter's second half, after guardrails).** `disallow flow pin -> log;` ‚Äî teach with the PIN/telemetry example; guardrails constrain actions, flow contracts constrain data; relate to `--redact` (read-side vs full lattice). *Gotcha:* flow is checked over FSL semantics ‚Äî a host hook that receives the secret is an *exit* from the proof; audit hook signatures. (¬ß10/¬ß17)
- **SLO contracts.** PCTL in contract position: the same property is the design-time proof (Storm backend) and the production drift monitor ‚Äî "your SLO is a contract on the machine, not a paragraph in a runbook." (¬ß17)
- **Composed-system checking.** What `finite` means for a *system* (all members finite + population `max` + queue bound ‚Äî finite **by construction**); the five checks (bounds, lifted contracts, deliverability, system `invariant` over `count`/`all`/`any`/`sum` aggregates, opt-in deadlock). *Teach:* **counterexamples are executable** ‚Äî the checker hands you a stimulus tape that `fsl run` replays; checker, stochastic tester, and factory are one machine. *Gotcha:* aggregates are system-context-only builtins. (¬ß23)

## Part VII ‚Äî Tooling & integration
- **Layout sugar `arranged`.** `arranged a -> b -> c;` = the chain *plus* `arrange [a b c]` ‚Äî DRY for the "write the sequence, then write the shape" annoyance. *Teach:* desugaring, decorations stay on edges, best for linear chains, repeats dedup; `arranged-start`/`arranged-end` analogs.
- **Template interpolation vs the expression language (teach the phase boundary).** `sm`/`ms` interpolation (`` sm`a ${x} -> b` ``) is **build-time host (JS) text-splicing** into the FSL source before parse (jssm.ts:5017) ‚Äî untyped, unhygienic, JS-only, and **gone after codegen** (Rust/C/Python have no tagged templates). The **expression language** (vals/sensors/guards) is **run-time, host-agnostic, typed, verifiable, replayable**. *Rule:* **interpolate to BUILD a machine** (generate structure from host data, machine families, host-computed constant tables ‚Äî the escape hatch the bounded expression language deliberately won't provide); **use a val/sensor/expression to make it BEHAVE dynamically** (probabilities, thresholds, defaults). *Gotchas:* interpolating a runtime value bakes one JS snapshot into source (loses typing/verification/replay/portability); interpolating untrusted input is **FSL injection** (string-injection into the grammar) ‚Äî use a typed val. *Verification symmetry:* build-time interpolation keeps a `finite` machine fully concrete for the checker; runtime vals/sensors add the input dimension it must over-approximate. (¬ß14/¬ß15)

## Appendix ‚Äî Gotchas / migration checklist (collect these for a "things that will surprise you" page)
- `var`‚Üí`val`, `property`‚Üí`prop` (alias removed v7); `set` keyword ‚Üí `assign`.
- Bracket-index **throws**; `get` returns `option`. `null = undefined` is **false**; no `null` in arithmetic.
- `any` is **sound-narrow-only** (must `case`). Bitwise on **sized** ints only.
- Units: **torque ‚â° energy** (use branding), **affine** temperature, **KB‚âÝKiB**.
- The **microstep bound** (default 100,000/reaction) catches eventless-loop hangs.
- Multigraph: probabilistic vs unlabeled distinction; `transition(to)` needs an unlabeled edge.
- Reserved words are **context-scoped** (your state named `case` still works outside expressions).
- `finite` machines **forbid** the rich tier (float/double, longint, open containers, recursive ADTs, first-class lambdas, `any`, unbounded) but **allow** over-approximated `rand`/`sensor`; `data` forbidden.
- SVG ids are **slugified** ‚Äî fetch them from the API.
- Compression encoders are **pinned** (don't expect host-native zlib byte-compat).
- Factory `seq()` indices are **never reused** after `retire`; `make()` moves the cursor, `at(n)` doesn't.
- Unseeded constructions **still have a seed** (clock-sampled once at the root, recorded) ‚Äî read it back with `.seed()` before claiming irreproducibility.
- `on_undeliverable` defaults to **error**; if your park "randomly crashes," a route is delivering to a state that doesn't accept the action ‚Äî `dead_letter` is the diagnostic mode.
- System containment must be **acyclic**; wiring may cycle. If the compiler rejects your composition, you wanted `wire`, not a member.
- GoF update: with document factories + `spawn`, **Factory Method / Abstract Factory are no longer out of domain** ‚Äî revise the Part-I GoF list when writing it.

StateKind

StateKind: "final" | "complete" | "terminal" | "base"
  • Defined in src/ts/jssm_viz.ts:576

Classification of a state for viz styling purposes. Each bucket is meaningfully distinct:

  • final — the state is both terminal AND complete (the strongest form; user-marked complete and has no exits).
  • complete — the state is user-marked complete but still has exits.
  • terminal — the state has no exits but is not user-marked complete (automatic / structural terminus).
  • base — none of the above.

Historical note: the viz code used to check state_is_final first, which made the complete and terminal branches structurally unreachable (because state_is_final in the jssm core is itself terminal || complete). That meant per-state-kind styling differences were silently absent for any state that was only complete or only terminal — a long-standing bug. The fix is to check the two underlying predicates directly so the three named buckets reflect three meaningfully distinct conditions.

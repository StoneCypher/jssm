Overlapping groups

An HTTP request is simultaneously "in progress" and "receiving" — two orthogonal axes of membership that don't fit a tree. Overlapping groups let one state belong to several groups at once, so a transition or a boundary hook can target a whole axis instead of every member by hand.

receiving is in both groups at once — it is in-progress and receiving. &InProgress 'abort' → idle expands to one abort edge per member, so any in-flight state can bail out without you writing five near-identical lines. on enter/on exit &Receiving fire only when a transition crosses the group boundary, never on moves that stay inside it.
